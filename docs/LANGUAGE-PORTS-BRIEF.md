## Source of truth (read first)
- **`NAMZ-FORMAT.md`** — the wire format: magic `NAMZ`, 1-byte version/codec/dtype/flags, `u16` metaLen +
  meta JSON, then body = `u32` skeletonLen + minified JSON skeleton (each numeric `"weights"` array
  replaced by its ordinal integer index) + `u32` numArrays + `u32` lengths[] + float32 payload. Integer
  fields are little-endian; the float payload is IEEE-754 little-endian. flags bit0 = payload byte-plane
  shuffled.
- **`include/namz.h`** — the C++17 reference impl (MIT). **`cli/namz.cpp`** — the CLI (`encode`/`decode`/
  `map`/`verify`).
- **`conformance/`** — the CROSS-LANGUAGE CONTRACT (a TCK): `vectors/*.nam` → `vectors/*.namz` golden
  pairs, `invalid/*.namz` must-reject blobs, and `manifest.json`. See `conformance/README.md`. This
  already exists; regenerate with `python3 conformance/generate.py build/namz` if you change the format.

## What .namz is
A `.nam` is JSON whose weights are ~20-char decimals the engine loads as float32 anyway. `.namz` stores
each weight as a 4-byte float32: ~5.5× smaller, bit-exact to float32, deterministic (no compression →
byte-identical across runs / platforms / languages). A readable metadata header exposes tone/device/gear
without decoding weights.

## Testing model (the whole point)
Every port runs the **shared conformance vectors**:
1. `encode(input, options)` **byte-matches** the expected `.namz`;
2. `decode(expected)` reproduces weights **bit-exact** (float32);
3. every `invalid/*.namz` is **rejected** cleanly (never crash/hang).

Plus a **local property fuzz** per language (N random finite float32 → round-trip → bit-exact) and
idiomatic unit tests — but the shared vectors are the guarantee.

### Byte-match gotchas
- Skeleton = **minified JSON, object keys sorted ascending** (nlohmann sorts). Python:
  `json.dumps(obj, sort_keys=True, separators=(",", ":"))`. JS: `JSON.stringify` doesn't sort → custom
  sorted+compact stringify. Where a language can't match key order, rely on decode-bit-exact +
  idempotence and document it — never fake a golden.
- Endianness: write `u16`/`u32` little-endian explicitly. Float payload IEEE-754 LE (numpy
  `astype("<f4").tobytes()`).
- Reproduce the byte-plane **shuffle** exactly (`shuffleInto`/`unshuffleInto` in `namz.h`).

## Per-language deliverable

vectors + local fuzz → packaging for the registry → CI (build + test + conformance, cross-OS where cheap)
→ publish-on-tag automation → README → **MIT** license.

## Versioning
The C++ reference is **v1.0.0**. **Each language port versions INDEPENDENTLY** — its own semver, its own
tag namespace, its own release cadence and registry. (Python `namz` on PyPI, JS `namz` on npm, etc. each
start at their own 1.0.0 when ready.) Do not couple a port's version to the C++ version; couple to the
**format/spec** version instead (the wire `formatVersion`, currently 2).
